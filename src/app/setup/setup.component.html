Setup Guide Setup a Node development environment You need Node.js, since the libraries required for Angular are downloaded using node package manager (npm) . Node.js powers client development and build tools. The npm package manager, which is itself a Node.js application, installs JavaScript libraries.

Visit below url and install node js
https://nodejs.org/en/

Verify that you are running Node.js v8.x or higher and npm 5.x or higher by running the commands node -v and npm -v in a terminal/console window. Older versions produce errors.

node -v

 Setup the Angular CLI What is Angular CLI

Angular CLI stands for Angular Command Line Interface. As the name implies, it is a command line tool for creating angular apps.

Using and Installing Angular CLI

Angular CLI is a command line interface for Angular, and is very useful in quickly creating an Angular 6 project template. Install the Angular CLI npm package globally using the following command:

npm install -g @angular/cli

npm Package manager will install Angular CLI for you.

 Create an Angular application The CLI creates a standard structure for project, and setup a working build. The build needs dependencies, that will be downloaded by default using the npm package manager.

Create an Angular application ng new my-app-name

*my-app-name is the name of app which you want to create.

This is going to take a while, but it will create a new project structure and it will install all the needed dependencies in one go. You now have a ready to use project!

Now to run application, go inside the app folder
cd my-app-name

Now run the serve command which will build the application ng serve

The ng serve command should start a development server on your locahost port 4200, so if you go to your browser and enter the following url:

http://localhost:4200

That's it, you have your first angular application ready.

 Understand Angular app structure When you create the project, you’ll notice that it creates a bunch of files. Here is the list of the important files and folders that you should be aware of:

In Visual Studio code, app folder structure will look like below -

  • package.json: This file has the list of node dependencies which are needed.

  • src/styles.css: This file has the global CSS available throughout the application.

  • src/main.: This is the main file which starts the Angular Application (AppModule is bootstrapped here as seen in the code ). Here the Extension .ts stands for TypeScript.

  • src/index.html: This the first file which executes alongside main.ts when the page loads.

  • src/app/app.module.ts: This is the file where all the components, providers, and modules are defined. Without defining them here, they can’t be used elsewhere in the code.

  • src/app/app.component.html: This is the main component of an angular app, and all other components are usually present within this component. src/app/app.component.ts is the logic for this component, and src/app/app.component.css is the CSS for this component. This component itself does not do any important logic, but acts as a container for other components.

  • dist: This folder is where the built files are present. TypeScript is basically converted to JavaScript and the resulting files are stored here after bundling and minification. (This Folder appears only if the application is built. A simple “npm start” will not create this folder. ) Since web browsers understand only JavaScript (at least for now), it is therefore necessary to convert TypeScript to JavaScript before deploying the code. To see this folder, you can type the following in the command prompt: npm run build

    • There are several other files as well, but knowing these basic ones is good for a start.

 Useful Angular CLI command You need a new component, module, service or any other angular construct. Angular-cli can create these files for you.

ng generate

if you need a new component, you could open a terminal at the desired location and use the below command to generate all the files you need to create a new component.

ng generate component [name]

The CLI will generate a new folder for you. If you don't want that extra folder, you can use the --flat option: ng generate component [name] --flat

This does not only work for components. You can create a whole bunch of different angular-blueprints by replacing component by one of the following.

  • class
  • directive
  • enum
  • guard
  • interface
  • module
  • pipe
  • service

Running your App

Once you are done creating and developing angular application, you also want to test and run it.

ng serve

It will start application in development mode. It will also watch the directory of your application for changes. So if it detects a change, it will recompile the application (partly) and automatically update the application in the browser.

Building

When you are happy with your angular app, it is time to deploy it to a web-server. To get the files, that are required for that, you can use the below command

ng build

This command will cause the CLI to build your application and places the output in a directory called "dist" (by default).

Using this command, the app is still in an development state.

This means, that certain optimizations where not performed and the app is still using the development environment variables.

To change that, use the --prod flag like this:

ng build --prod

Disable the CLI for your project

ng eject

It will disable the CLI for your project and create you a webpack configuration file.

While the CLI was using webpack under the hood anyway, you now have full control over webpack.

Keep in mind, that this command is a one way ticket. There is no command that coverts the project pack to angular-cli-mode.
